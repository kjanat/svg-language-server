# Progress Log

PRD: full-snapshot-svg-data
Started: 2026-04-09

## Codebase Patterns

- Canonical truth must be stored per snapshot, not as union-plus-exceptions.
- Union data and version overlays are derived artifacts only.
- SVG 1.1 snapshots rely on TR indices plus flattened DTDs for structure.
- SVG 2 CR uses W3C TR as authority even when GitHub assists extraction.
- SVG 2 ED should parse the pinned `w3c/svgwg` commit and `definitions*.xml` files directly.
- Raw fetched source bundles stay out of git; pinned manifests and normalized data stay in git.
- Value grammar is first-class data, not only display text.
- External specs start as pinned typed foreign references, not full ingest.
- Source acquisition is staged as checked-in TOML manifests under `crates/svg-data/data/sources/`, one file per snapshot or external spec set.
- Snapshot schemas should use serde-tagged enums plus canonical `SpecSnapshotId` values so checked-in JSON stays explicit and stable.
- Shared snapshot ingestion should flow through four reusable steps: typed manifest load, cache-backed fetch, manifest-derived provenance, and stable per-snapshot JSON emission.
- Seed new checked-in snapshot datasets from the current profile-aware catalog first, then freeze parity with a dataset-specific regression test before replacing the seed with source-native parsing.
