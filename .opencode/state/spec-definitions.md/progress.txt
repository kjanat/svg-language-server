## Codebase Patterns

- Keep `svg-data` public profile metadata in `src/lib.rs` + `src/types.rs` first when it is small and static; save build-script churn for later profile-specific catalog overlays.
- Normalize user-facing identifiers by stripping punctuation and folding ASCII case when config should accept canonical ids, aliases, and human-written long-form names.
- Keep spec lifecycle as its own field on `svg-data` defs; compat booleans stay a separate axis so later lint/LSP code can switch without re-plumbing catalog generation again.
